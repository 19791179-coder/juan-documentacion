### 🔍 Investigación de Normas

#### IEEE 829 — Documentación de Pruebas de Software

1. Documentación clara y comprensible para todo el equipo
2. Registro completo de pruebas (fecha, responsable, entorno)
3. Resultados esperados para cada caso de prueba
4. Procedimientos paso a paso con condiciones previas
5. Evidencia documentada de errores con capturas

#### ISO/IEC 26514 — Documentación para Usuarios

1. Lenguaje apropiado al nivel técnico del lector
2. Organización lógica por secciones y subsecciones
3. Imágenes, capturas y diagramas ilustrativos
4. Instrucciones paso a paso numeradas
5. Autosuficiencia del manual sin asistencia adicional

- ✅ **Resolución mínima:** 1280 × 720 píxeles
- ✅ **Formato:** PNG (compresión sin pérdida)

## 📖 Glosario para Usuario Final

| Término Técnico | Significado (para el usuario) |
|-----------------|-------------------------------|
| **API** | "El mesero del sistema: pide información y la trae" |
| **Backup** | Copia de seguridad de los datos |
| **CRUD** | Crear, Leer, Actualizar, Eliminar — acciones básicas |
| **Dashboard** | Panel de control con información resumida |
| **Deploy** | "Abrir la tienda después de renovarla" |
| **Foreign Key** | "El enlace entre dos listas" |
| **Frontend** | La parte que el usuario ve e interactúa |
| **Login** | Inicio de sesión con usuario y contraseña |
| **Query** | "Pregunta que le hacemos al sistema" |
| **Rollback** | "El botón deshacer del sistema" |
| **Trigger** | "Regla automática que se activa al guardar" |
| **Validación** | Verificación de datos antes de guardar |

### Ejemplo de Simplificación

| Frase original | Frase reescrita |
|----------------|-----------------|
| *"El trigger de la FK disparó una violación de constraint al intentar insertar el registro."* | *"El sistema no pudo guardar la información porque detectó un dato incorrecto. Revise los campos marcados con (*) y vuelva a intentarlo."* |
